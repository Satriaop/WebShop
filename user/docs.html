Otentikasi

Semua permintaan API memerlukan parameter key berisi API Key kamu. Bersifat rahasia — jangan dibagikan atau disematkan di kode yang bisa diakses publik.

 Daftar Endpoint

  • GET /api/profile — Informasi Akun
  • GET /api/listkategori — Daftar Kategori Produk
  • GET /api/listproduk — Daftar Produk & Stok
  • POST /api/order — Buat Pesanan

Akun

GET /api/profile

Mengambil detail informasi akun yang terhubung dengan API Key.

Parameter Query

key
String (Required)
API Key unik kamu untuk memverifikasi identitas permintaan.

Respon Berhasil (200 OK)

{
  "status": true,
  "data": {
    "username": "dikadev",
    "phone": "083121495921",
    "role": "owner",
    "apiKey": "sk-0277..."
  }
}

Produk

GET /api/listkategori

Mengambil seluruh daftar kategori produk yang tersedia.

Parameter Query

key
String (Required)
API Key kamu.
Jika belum ada produk, endpoint ini tetap mengembalikan 200 OK dengan data: [] — tidak akan 404.

Respon Berhasil (200 OK)

{
  "status": true,
  "total": 3,
  "data": [
    "GAME",
    "STREAMING",
    "VOUCHER"
  ]
}

Respon Kosong (200 OK)

{
  "status": true,
  "total": 0,
  "data": []
}
GET /api/listproduk

Mengambil daftar produk beserta jumlah stok yang tersedia. Parameter kategori bersifat opsional — jika tidak diisi, semua produk dikembalikan.

Parameter Query

key
String (Required)
API Key kamu.
kategori
String (Optional)
Filter produk berdasarkan kategori (contoh: GAME, STREAMING). Tidak case-sensitive. Jika dikosongkan, semua produk dikembalikan.
Jika tidak ada produk yang cocok, endpoint mengembalikan 200 OK dengan data: [] — bukan 404.
Field stok pada setiap produk menunjukkan jumlah unit yang masih tersedia saat ini.

Respon Berhasil (200 OK)

{
  "status": true,
  "kategori": "GAME",
  "total": 2,
  "data": [
    {
      "productId": "prod_abc123",
      "nama": "Mobile Legends 100 Diamond",
      "harga": 20000,
      "kategori": "GAME",
      "stok": 15,
      "deskripsi": "Top up diamond ML"
    }
  ]
}

Contoh Request

# Semua produk
GET /api/listproduk?key=sk-xxxx

# Filter per kategori
GET /api/listproduk?key=sk-xxxx&kategori=GAME

Transaksi

POST /api/order

Membuat pesanan baru. Cukup kirim API Key beserta detail produk — tidak perlu token tambahan.

Gunakan metode POST dengan body JSON. Pastikan header Content-Type: application/json disertakan.

Request Body (JSON)

key
String (Required)
API Key kamu sebagai identitas & otorisasi.
productId
String (Required)
ID produk yang ingin dibeli (dari endpoint /api/listproduk).
jumlah
Number (Required)
Jumlah unit yang dipesan. Harus lebih dari 0 dan tidak melebihi stok tersedia.
catatan
String (Optional)
Catatan tambahan untuk pesanan (misal: user ID game, nomor akun, dll).

Contoh Request

POST /api/order
Content-Type: application/json

{
  "key": "sk-xxxxxxxxxxxx",
  "productId": "prod_abc123",
  "jumlah": 1,
  "catatan": "UserID: 123456789"
}

Respon Berhasil (200 OK)

{
  "status": true,
  "message": "pesanan berhasil dibuat",
  "data": {
    "orderId": "ord_20260510_xyz",
    "productId": "prod_abc123",
    "jumlah": 1,
    "total": 20000,
    "status": "pending",
    "createdAt": "2026-05-10T10:30:00.000Z"
  }
}

Respon Error

// 401 — API Key tidak valid
{ "status": false, "message": "api key tidak valid" }

// 400 — Produk tidak ditemukan
{ "status": false, "message": "produk tidak ditemukan" }

// 400 — Stok tidak cukup
{ "status": false, "message": "stok tidak mencukupi" }

// 500 — Server error
{ "status": false, "message": "terjadi kesalahan pada server" }
